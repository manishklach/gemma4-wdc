# Roadmap

## Near Term

- strengthen the one-node hybrid path for a real local Gemma-family adapter
- add replayable task logs for demos and benchmark inspection
- make collapse decisions more explainable in the dashboard
- deepen the coding-agent scenario with richer repo-analysis tasks

## Mid Term

- branch-aware intermediate state reuse beyond full result fan-out
- stronger per-task-type thresholds and verifier hooks
- better local document and code backends for more realistic demos
- trace-driven benchmarks in addition to hand-authored scenarios

## Longer Term

- true multi-node mode
- heterogeneous Gemma-family nodes
- distributed lock synchronization
- stronger semantic matching
- persistent runtime state and replay
