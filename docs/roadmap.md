# Roadmap

## Near-Term

- explainable `why collapsed?` and `why not collapsed?` paths in the UI and API
- stronger benchmark artifacts and repeatable benchmark summaries
- richer coding-agent scenarios for repo scan, code search, and test-planning overlap
- more explicit threshold tuning and task fingerprint debugging

## Mid-Term

- distributed coordination for SEU creation and execution ownership
- branch-aware intermediate state reuse beyond full result fan-out
- verifier hooks for coding-agent fan-out safety
- trace capture and trajectory replay for benchmark reproducibility

## Longer-Term

- execution-aware scheduling hints back to branch planners
- shared artifact invalidation for branch lineage changes
- state reuse across long-horizon runs
- multi-runtime trace analysis and branch divergence tooling
