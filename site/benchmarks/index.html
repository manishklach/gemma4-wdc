Benchmarks

Preliminary results that show behavior clearly rather than overselling performance.

The current harness is scenario-driven and uses mock executors. That makes the benchmark numbers intentionally modest, transparent, and easy to audit.

Scenario families

  • coding_repo_scan: overlapping repo-understanding tasks from concurrent coding branches
  • document_research: repeated evidence extraction across the same corpus
  • api_fanout: overlapping outbound API operations
  • false_collapse_safety: similar-looking tasks that should remain separate

Methodology notes

The benchmark harness compares naive execution against shared execution and reports tasks received, shared execution units created, executions saved, dedup ratio, and false-collapse rate. It is meant to be honest and inspectable, not flashy.
