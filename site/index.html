Agent Runtime Lab

One concrete runtime answer to duplicated subwork in concurrent agent systems.

Shared Execution Runtime detects semantic overlap across branch-local tasks, opens a bounded non-resetting admission window, collapses matching work into a shared execution unit, executes once, fans out the result, and measures the savings.

Read the README Run the demo Understand the architecture

Problem

Parallel agent branches increasingly repeat semantically overlapping work.

As systems move from single-threaded chat toward concurrent branch execution, duplicated repo scans, repeated code search, repeated evidence extraction, and repeated API operations become a runtime bottleneck.

Core Idea

Detect overlap early, delay briefly, execute once, fan out explicitly.

This repo explores a branch-aware runtime layer rather than another app shell. The emphasis is on task fingerprinting, admission control, shared execution units, and observability.
